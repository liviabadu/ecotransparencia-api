# US-001: Buscar entidade por CNPJ

## Descrição

**Como** um usuário do sistema
**Quero** pesquisar uma empresa pelo seu CNPJ
**Para** verificar se ela possui áreas embargadas pelo IBAMA

## Contexto

Esta é a funcionalidade principal de consulta do sistema EcoTransparência. Permite que usuários verifiquem se uma empresa (pessoa jurídica) possui registros de áreas embargadas pelo IBAMA, indicando possíveis irregularidades ambientais.

## Endpoint

```
GET /api/search/document?document={cnpj}&type=cnpj
```

### Parâmetros

| Parâmetro | Tipo   | Obrigatório | Descrição                        |
|-----------|--------|-------------|----------------------------------|
| document  | String | Sim         | CNPJ da empresa (14 dígitos)     |
| type      | String | Sim         | Tipo do documento: "cnpj"        |

---

## Critérios de Aceitação

### AC-01: Aceitar CNPJ como parâmetro de busca
- **Dado** que o usuário informa um CNPJ
- **Quando** o sistema recebe a requisição
- **Então** deve aceitar o parâmetro `document` com 14 dígitos numéricos
- **E** o parâmetro `type` deve ser "cnpj"

**Teste:** Validação de parâmetros na requisição

### AC-02: Retornar entidade encontrada
- **Dado** que existe uma empresa com CNPJ `11222333000181` na base de dados
- **Quando** o usuário busca por este CNPJ
- **Então** o sistema deve retornar status HTTP 200
- **E** o campo `found` deve ser `true`
- **E** o objeto `entity` deve conter os dados da empresa

**Teste de Contrato:** `a request to search by CNPJ` com state `an entity with CNPJ 11222333000181 exists`

### AC-03: Estrutura da resposta (entidade encontrada)
- **Dado** que a entidade foi encontrada
- **Então** a resposta deve conter:
  - `found`: boolean (true)
  - `entity.id`: String - identificador único (SEQ_TAD)
  - `entity.name`: String - nome da empresa (NOME_PESSOA_EMBARGADA)
  - `entity.document`: String - CNPJ (CPF_CNPJ_EMBARGADO)
  - `entity.documentType`: String - "cnpj"
  - `entity.riskLevel`: String - nível de risco calculado
  - `entity.score`: Integer - pontuação de risco (0-100)
  - `entity.occurrences`: Array - lista de embargos

**Teste:** Validação da estrutura JSON de resposta

### AC-04: Estrutura das ocorrências
- **Dado** que a entidade possui embargos
- **Então** cada ocorrência deve conter:
  - `id`: String - identificador do embargo
  - `category`: String - tipo de área embargada
  - `date`: String - data do embargo (ISO 8601)
  - `description`: String - descrição do embargo
  - `source`: String - "IBAMA"
  - `sourceUrl`: String - URL para consulta
  - `status`: String - "Ativo" ou "Baixado"

**Teste:** Validação da estrutura de ocorrências

### AC-05: Retornar não encontrado
- **Dado** que não existe uma empresa com o CNPJ informado
- **Quando** o usuário busca por um CNPJ inexistente
- **Então** o sistema deve retornar status HTTP 200
- **E** o campo `found` deve ser `false`
- **E** o objeto `entity` NÃO deve estar presente

**Teste de Contrato:** `a request to search by non-existent CNPJ` (US-004)

## Mapeamento de Campos (IBAMA → API)

| Campo API                | Campo IBAMA              | Transformação                    |
|--------------------------|--------------------------|----------------------------------|
| entity.id                | SEQ_TAD                  | String                           |
| entity.name              | NOME_PESSOA_EMBARGADA    | String                           |
| entity.document          | CPF_CNPJ_EMBARGADO       | String (apenas números)          |
| entity.documentType      | -                        | "cnpj" (fixo)                    |
| entity.riskLevel         | -                        | Calculado pelo score             |
| entity.score             | -                        | Calculado (0-100)                |
| occurrence.id            | SEQ_TAD                  | String                           |
| occurrence.category      | TP_AREA_EMBARGADA        | String                           |
| occurrence.date          | DAT_EMBARGO              | ISO 8601                         |
| occurrence.description   | DES_TAD                  | String                           |
| occurrence.source        | -                        | "IBAMA" (fixo)                   |
| occurrence.sourceUrl     | -                        | URL construída                   |
| occurrence.status        | -                        | "Ativo" ou "Baixado"             |

## Regras de Negócio

### RN-01: Cálculo do Score de Risco
O score é calculado considerando:
- Quantidade de embargos ativos (+10 por embargo)
- Área total embargada em hectares
- Recência dos embargos (mais recentes = maior peso)
- Tipo de infração (desmatamento = maior peso)

### RN-02: Classificação do Nível de Risco
| Score   | Nível de Risco |
|---------|----------------|
| 0-25    | Baixo          |
| 26-50   | Médio          |
| 51-79   | Alto           |
| 80-100  | Crítico        |

### RN-03: Agregação de Embargos
Quando uma empresa possui múltiplos embargos, todos devem ser listados no array `occurrences`, ordenados por data (mais recentes primeiro).
