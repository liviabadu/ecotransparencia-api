# US-004: Exibir entidade não encontrada

## Descrição

**Como** um usuário do sistema
**Quero** receber feedback claro quando uma entidade não for encontrada
**Para** saber que a busca foi executada mas não há registros de embargo

## Contexto

Esta funcionalidade garante uma experiência consistente quando o usuário pesquisa por uma entidade (CPF, CNPJ ou nome) que não possui registros de embargo no sistema. Em vez de retornar um erro HTTP 404, o sistema retorna HTTP 200 com `found: false`, indicando que a busca foi bem-sucedida mas não encontrou resultados.

## Endpoints Afetados

Todos os endpoints de busca:
```
GET /api/search/document?document={cnpj}&type=cnpj
GET /api/search/document?document={cpf}&type=cpf
GET /api/search/name?name={termo}
```

---

## Critérios de Aceitação

### AC-01: Retornar HTTP 200 mesmo quando não encontrado
- **Dado** que o usuário busca por um documento ou nome
- **Quando** não existe entidade correspondente na base de dados
- **Então** o sistema deve retornar status HTTP 200
- **E** NÃO deve retornar erro 404

**Teste de Contrato:** `a request to search by non-existent CNPJ`

### AC-02: Retornar found: false
- **Dado** que a entidade não foi encontrada
- **Quando** o sistema monta a resposta
- **Então** o campo `found` deve ser `false`

**Teste:** Validação do campo found na resposta

### AC-03: Não incluir objeto entity
- **Dado** que a entidade não foi encontrada
- **Quando** o sistema monta a resposta
- **Então** o objeto `entity` NÃO deve estar presente na resposta
- **E** a resposta deve conter apenas o campo `found`

**Teste de Contrato:** Validação da estrutura JSON sem entity

### AC-04: Funcionar para busca por CNPJ inexistente
- **Dado** que o usuário busca por um CNPJ que não existe
- **Quando** não há registros com esse CNPJ
- **Então** deve retornar `{"found": false}`

**Teste de Contrato:** `a request to search by non-existent CNPJ` com state `no entity with CNPJ 00000000000000 exists`

### AC-05: Funcionar para busca por CPF inexistente
- **Dado** que o usuário busca por um CPF que não existe
- **Quando** não há registros com esse CPF
- **Então** deve retornar `{"found": false}`

**Teste:** Reutiliza lógica do AC-04

### AC-06: Funcionar para busca por nome inexistente
- **Dado** que o usuário busca por um nome que não existe
- **Quando** não há registros contendo o termo
- **Então** deve retornar `{"found": false}`

**Teste de Contrato:** `a request to search by non-existent name` com state `no entity with name "Entidade Inexistente XYZ" exists`

## Modelo de Dados

### Response (Não Encontrado - 200 OK)

```json
{
  "found": false
}
```

### Comparação com Response de Sucesso

| Cenário           | HTTP Status | found  | entity   |
|-------------------|-------------|--------|----------|
| Encontrado        | 200         | true   | presente |
| Não encontrado    | 200         | false  | ausente  |

---

## Regras de Negócio

### RN-01: HTTP 200 para todos os cenários de busca
O sistema sempre retorna HTTP 200 para buscas válidas, independente de encontrar resultados ou não. Isso diferencia:
- **200 + found: true** → Entidade encontrada com dados
- **200 + found: false** → Busca executada, sem resultados
- **400** → Parâmetros inválidos (futuro)
- **500** → Erro interno do servidor

### RN-02: Resposta minimalista quando não encontrado
A resposta para entidade não encontrada deve ser minimalista, contendo apenas:
```json
{
  "found": false
}
```

Não deve incluir campos vazios, nulls ou objetos entity vazios.

### RN-03: Consistência entre endpoints
O comportamento de "não encontrado" deve ser idêntico para:
- Busca por CNPJ (`/api/search/document?type=cnpj`)
- Busca por CPF (`/api/search/document?type=cpf`)
- Busca por nome (`/api/search/name`)

## Implementação

### SearchResponse.java

```java
public class SearchResponse {
    private boolean found;
    private EntityDto entity;

    public static SearchResponse notFound() {
        SearchResponse response = new SearchResponse();
        response.setFound(false);
        // entity permanece null e não é serializado
        return response;
    }

    public static SearchResponse found(EntityDto entity) {
        SearchResponse response = new SearchResponse();
        response.setFound(true);
        response.setEntity(entity);
        return response;
    }
}
```

### SearchService.java

```java
public SearchResponse searchByDocument(String document, String type) {
    List<Embargo> embargos = embargoRepository.findByDocument(document);

    if (embargos.isEmpty()) {
        return SearchResponse.notFound();  // US-004
    }

    EntityDto entity = buildEntityDto(embargos, document, type);
    return SearchResponse.found(entity);
}
```
