# US-002: Buscar entidade por CPF

## Descrição

**Como** um usuário do sistema
**Quero** pesquisar uma pessoa física pelo seu CPF
**Para** verificar se ela possui áreas embargadas pelo IBAMA

## Contexto

Esta funcionalidade permite a consulta de pessoas físicas que possuem registros de áreas embargadas pelo IBAMA. É análoga à US-001 (busca por CNPJ), mas focada em indivíduos. Embargos de pessoas físicas frequentemente estão relacionados a propriedades rurais com desmatamento ilegal ou outras infrações ambientais.

## Endpoint

```
GET /api/search/document?document={cpf}&type=cpf
```

### Parâmetros

| Parâmetro | Tipo   | Obrigatório | Descrição                          |
|-----------|--------|-------------|------------------------------------|
| document  | String | Sim         | CPF da pessoa física (11 dígitos)  |
| type      | String | Sim         | Tipo do documento: "cpf"           |

---

## Critérios de Aceitação

### AC-01: Aceitar CPF como parâmetro de busca
- **Dado** que o usuário informa um CPF
- **Quando** o sistema recebe a requisição
- **Então** deve aceitar o parâmetro `document` com 11 dígitos numéricos
- **E** o parâmetro `type` deve ser "cpf"

**Teste:** Validação de parâmetros na requisição

### AC-02: Retornar pessoa encontrada
- **Dado** que existe uma pessoa com CPF `12345678909` na base de dados
- **Quando** o usuário busca por este CPF
- **Então** o sistema deve retornar status HTTP 200
- **E** o campo `found` deve ser `true`
- **E** o objeto `entity` deve conter os dados da pessoa

**Teste de Contrato:** `a request to search by CPF` com state `a person with CPF 12345678909 exists`

### AC-03: Estrutura da resposta (pessoa encontrada)
- **Dado** que a pessoa foi encontrada
- **Então** a resposta deve conter:
  - `found`: boolean (true)
  - `entity.id`: String - identificador único (SEQ_TAD)
  - `entity.name`: String - nome da pessoa (NOME_PESSOA_EMBARGADA)
  - `entity.document`: String - CPF (CPF_CNPJ_EMBARGADO)
  - `entity.documentType`: String - "cpf"
  - `entity.riskLevel`: String - nível de risco calculado
  - `entity.score`: Integer - pontuação de risco (0-100)
  - `entity.occurrences`: Array - lista de embargos

**Teste:** Validação da estrutura JSON de resposta

### AC-04: Estrutura das ocorrências
- **Dado** que a pessoa possui embargos
- **Então** cada ocorrência deve conter:
  - `id`: String - identificador do embargo
  - `category`: String - tipo de área embargada
  - `date`: String - data do embargo (ISO 8601)
  - `description`: String - descrição do embargo
  - `source`: String - "IBAMA"
  - `sourceUrl`: String - URL para consulta
  - `status`: String - "Ativo" ou "Baixado"

**Teste:** Validação da estrutura de ocorrências

### AC-05: Retornar não encontrado
- **Dado** que não existe uma pessoa com o CPF informado
- **Quando** o usuário busca por um CPF inexistente
- **Então** o sistema deve retornar status HTTP 200
- **E** o campo `found` deve ser `false`
- **E** o objeto `entity` NÃO deve estar presente

**Teste de Contrato:** Reutiliza cenário de US-004 (entidade não encontrada)

### AC-06: Indicar infrações de desmatamento
- **Dado** que a pessoa possui embargo relacionado a desmatamento
- **Quando** o campo `SIT_DESMATAMENTO` for "D"
- **Então** o score de risco deve ter peso adicional (+5 pontos)
- **E** isso deve ser refletido no cálculo do `riskLevel`

**Teste:** Validação do cálculo de risco com desmatamento

## Mapeamento de Campos (IBAMA -> API)

| Campo API                | Campo IBAMA              | Transformação                    |
|--------------------------|--------------------------|----------------------------------|
| entity.id                | SEQ_TAD                  | String                           |
| entity.name              | NOME_PESSOA_EMBARGADA    | String                           |
| entity.document          | CPF_CNPJ_EMBARGADO       | String (apenas números)          |
| entity.documentType      | -                        | "cpf" (fixo)                     |
| entity.riskLevel         | -                        | Calculado pelo score             |
| entity.score             | -                        | Calculado (0-100)                |
| occurrence.id            | SEQ_TAD                  | "occ-" + SEQ_TAD                 |
| occurrence.category      | TP_AREA_EMBARGADA        | String                           |
| occurrence.date          | DAT_EMBARGO              | ISO 8601                         |
| occurrence.description   | DES_TAD                  | String                           |
| occurrence.source        | -                        | "IBAMA" (fixo)                   |
| occurrence.sourceUrl     | -                        | URL construída                   |
| occurrence.status        | -                        | "Ativo" ou "Baixado"             |

## Regras de Negócio

### RN-01: Validação do CPF
- O CPF deve conter exatamente 11 dígitos numéricos
- Caracteres especiais (pontos, traços) devem ser removidos antes da busca
- O sistema NÃO valida o dígito verificador (apenas o formato)

### RN-02: Cálculo do Score de Risco
O score é calculado considerando:
- **+10 pontos** por cada embargo ativo
- **+5 pontos** por embargo relacionado a desmatamento (SIT_DESMATAMENTO = 'D')
- Score máximo limitado a 100

### RN-03: Classificação do Nível de Risco
| Score   | Nível de Risco |
|---------|----------------|
| 0-25    | Baixo          |
| 26-50   | Médio          |
| 51-79   | Alto           |
| 80-100  | Crítico        |

### RN-04: Agregação de Embargos
Quando uma pessoa possui múltiplos embargos, todos devem ser listados no array `occurrences`, ordenados por data (mais recentes primeiro).

### Teste de Contrato (Pact)

**Arquivo:** `src/test/java/br/com/ecotransparencia/contract/ProviderPactVerificationTest.java`

**Interação:** `a request to search by CPF`

**Provider State:** `a person with CPF 12345678909 exists`

```java
@State("a person with CPF 12345678909 exists")
void setupPersonWithCpf() {
    QuarkusTransaction.requiringNew().run(() -> {
        EmbargoRepository repository = getRepository();
        repository.deleteAll();

        Embargo embargo = new Embargo();
        embargo.setSeqTad(1600976L);
        embargo.setNumTad("789209");
        embargo.setSerTad("A");
        embargo.setNomePessoaEmbargada("RAIMUNDO CIRINO DE SOUZA");
        embargo.setCpfCnpjEmbargado("12345678909");
        embargo.setTpAreaEmbargada("Desmatamento");
        embargo.setDatEmbargo(LocalDateTime.of(2020, 8, 15, 0, 0));
        embargo.setDesTad("Por ter desmatado aproximadamente 10,00 ha de mata nativa sem autorização.");
        embargo.setSigUfTad("MT");
        embargo.setNomMunicipioTad("Alta Floresta");
        embargo.setSitDesmatamento("D");
        embargo.setQtdAreaEmbargada(BigDecimal.valueOf(10.0));
        embargo.setNumAutoInfracao("789209");
        embargo.setSerAutoInfracao("A");
        embargo.setDesTipoBioma("Amazônia");
        embargo.setCodTipoBioma(4);

        repository.persist(embargo);
    });
}
```

## Diferenças em relação à US-001

| Aspecto              | US-001 (CNPJ)                  | US-002 (CPF)                       |
|----------------------|--------------------------------|------------------------------------|
| Tipo documento       | Pessoa jurídica                | Pessoa física                      |
| Formato              | 14 dígitos                     | 11 dígitos                         |
| Parâmetro `type`     | "cnpj"                         | "cpf"                              |
| Casos típicos        | Empresas, prefeituras          | Proprietários rurais, indivíduos   |
| Desmatamento         | Menos frequente                | Mais frequente                     |
