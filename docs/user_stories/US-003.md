# US-003: Buscar entidade por nome

## Descrição

**Como** um usuário do sistema
**Quero** pesquisar entidades pelo nome
**Para** encontrar empresas ou pessoas quando não possuo o documento (CPF/CNPJ)

## Contexto

Esta funcionalidade permite a busca de entidades embargadas através do nome, útil quando o usuário não possui o CPF ou CNPJ. A busca é realizada de forma parcial (contém o termo) e case-insensitive, permitindo encontrar entidades mesmo com digitação incompleta do nome.

## Endpoint

```
GET /api/search/name?name={termo}
```

### Parâmetros

| Parâmetro | Tipo   | Obrigatório | Descrição                              |
|-----------|--------|-------------|----------------------------------------|
| name      | String | Sim         | Termo de busca (nome parcial ou total) |

---

## Critérios de Aceitação

### AC-01: Aceitar termo de busca como parâmetro
- **Dado** que o usuário informa um termo de busca
- **Quando** o sistema recebe a requisição
- **Então** deve aceitar o parâmetro `name` com qualquer texto
- **E** o termo pode ser nome parcial ou completo

**Teste:** Validação de parâmetros na requisição

### AC-02: Realizar busca parcial case-insensitive
- **Dado** que existe uma entidade com nome "Empresa Verde Sustentável Ltda"
- **Quando** o usuário busca por "empresa verde" (minúsculas)
- **Então** o sistema deve encontrar a entidade
- **E** a busca deve ignorar diferenças de maiúsculas/minúsculas

**Teste de Contrato:** `a request to search by name` com state `an entity with name containing "Empresa Verde" exists`

### AC-03: Retornar entidade encontrada
- **Dado** que existe uma entidade com nome contendo o termo buscado
- **Quando** o usuário realiza a busca
- **Então** o sistema deve retornar status HTTP 200
- **E** o campo `found` deve ser `true`
- **E** o objeto `entity` deve conter os dados da primeira entidade encontrada

**Teste:** Validação da estrutura JSON de resposta

### AC-04: Estrutura da resposta (entidade encontrada)
- **Dado** que a entidade foi encontrada
- **Então** a resposta deve conter:
  - `found`: boolean (true)
  - `entity.id`: String - identificador único (SEQ_TAD)
  - `entity.name`: String - nome da entidade (NOME_PESSOA_EMBARGADA)
  - `entity.document`: String - CPF ou CNPJ (CPF_CNPJ_EMBARGADO)
  - `entity.documentType`: String - "cpf" ou "cnpj" (inferido pelo tamanho)
  - `entity.riskLevel`: String - nível de risco calculado
  - `entity.score`: Integer - pontuação de risco (0-100)
  - `entity.occurrences`: Array - lista de embargos

**Teste:** Validação da estrutura de resposta

### AC-05: Estrutura das ocorrências
- **Dado** que a entidade possui embargos
- **Então** cada ocorrência deve conter:
  - `id`: String - identificador do embargo
  - `category`: String - tipo de área embargada
  - `date`: String - data do embargo (ISO 8601)
  - `description`: String - descrição do embargo
  - `source`: String - "IBAMA"
  - `sourceUrl`: String - URL para consulta
  - `status`: String - "Ativo" ou "Baixado"

**Teste:** Validação da estrutura de ocorrências

### AC-06: Retornar não encontrado
- **Dado** que não existe entidade com nome contendo o termo buscado
- **Quando** o usuário realiza a busca
- **Então** o sistema deve retornar status HTTP 200
- **E** o campo `found` deve ser `false`
- **E** o objeto `entity` NÃO deve estar presente

**Teste de Contrato:** `a request to search by non-existent name` (US-004)

### AC-07: Inferir tipo de documento pelo tamanho
- **Dado** que a entidade foi encontrada
- **Quando** o documento possui 11 dígitos
- **Então** `documentType` deve ser "cpf"
- **Quando** o documento possui 14 dígitos
- **Então** `documentType` deve ser "cnpj"

**Teste:** Validação da inferência do tipo de documento

## Mapeamento de Campos (IBAMA -> API)

| Campo API                | Campo IBAMA              | Transformação                         |
|--------------------------|--------------------------|---------------------------------------|
| entity.id                | SEQ_TAD                  | String                                |
| entity.name              | NOME_PESSOA_EMBARGADA    | String                                |
| entity.document          | CPF_CNPJ_EMBARGADO       | String (apenas números)               |
| entity.documentType      | -                        | Inferido: 11 dígitos="cpf", 14="cnpj" |
| entity.riskLevel         | -                        | Calculado pelo score                  |
| entity.score             | -                        | Calculado (0-100)                     |
| occurrence.id            | SEQ_TAD                  | "occ-" + SEQ_TAD                      |
| occurrence.category      | TP_AREA_EMBARGADA        | String                                |
| occurrence.date          | DAT_EMBARGO              | ISO 8601                              |
| occurrence.description   | DES_TAD                  | String                                |
| occurrence.source        | -                        | "IBAMA" (fixo)                        |
| occurrence.sourceUrl     | -                        | URL construída                        |
| occurrence.status        | -                        | "Ativo" ou "Baixado"                  |

## Regras de Negócio

### RN-01: Busca parcial case-insensitive
- A busca deve utilizar `LIKE` com `%termo%`
- A comparação deve ignorar maiúsculas/minúsculas (lower/upper)
- Exemplo: buscar "verde" encontra "Empresa Verde Sustentável Ltda"

### RN-02: Inferência do tipo de documento
- Se `CPF_CNPJ_EMBARGADO` tem 11 caracteres → `documentType = "cpf"`
- Se `CPF_CNPJ_EMBARGADO` tem 14 caracteres → `documentType = "cnpj"`
- Outros tamanhos → `documentType = "unknown"`

### RN-03: Resultado único (primeira entidade)
- Quando múltiplas entidades correspondem ao termo, retornar apenas a primeira
- Entidades são agrupadas por `CPF_CNPJ_EMBARGADO`
- Todos os embargos da entidade são listados em `occurrences`

### RN-04: Cálculo do Score de Risco
O score é calculado considerando:
- **+10 pontos** por cada embargo ativo
- **+5 pontos** por embargo relacionado a desmatamento (SIT_DESMATAMENTO = 'D')
- Score máximo limitado a 100

### RN-05: Classificação do Nível de Risco
| Score   | Nível de Risco |
|---------|----------------|
| 0-25    | Baixo          |
| 26-50   | Médio          |
| 51-79   | Alto           |
| 80-100  | Crítico        |
