# US-005: Identificar entidades com risco crítico

## Descrição

**Como** um analista de compliance
**Quero** visualizar claramente entidades com nível de risco crítico
**Para** priorizar análises e tomadas de decisão

## Contexto

Esta funcionalidade aprimora o cálculo do score de risco das entidades, considerando múltiplos fatores como quantidade de embargos, área embargada, recência, tipo de infração (desmatamento) e bioma afetado (Amazônia). Entidades com score >= 80 são classificadas como "Crítico" e devem receber atenção prioritária.

## Endpoint

Reutiliza os endpoints existentes:
```
GET /api/search/document?document={cnpj}&type=cnpj
GET /api/search/document?document={cpf}&type=cpf
GET /api/search/name?name={termo}
```

## Critérios de Aceitação

### AC-01: Calcular score baseado na quantidade de embargos
- **Dado** que uma entidade possui múltiplos embargos
- **Quando** o sistema calcula o score de risco
- **Então** deve adicionar **+10 pontos** por cada embargo

**Teste:** Entidade com 8 embargos deve ter no mínimo 80 pontos base

### AC-02: Adicionar peso para embargos de desmatamento
- **Dado** que um embargo possui `SIT_DESMATAMENTO = 'D'`
- **Quando** o sistema calcula o score de risco
- **Então** deve adicionar **+5 pontos** extras por embargo de desmatamento

**Teste:** Embargo com desmatamento adiciona 15 pontos (10 base + 5 desmatamento)

### AC-03: Adicionar peso para bioma Amazônia
- **Dado** que um embargo ocorreu no bioma Amazônia (`COD_TIPO_BIOMA = 4`)
- **Quando** o sistema calcula o score de risco
- **Então** deve adicionar **+3 pontos** extras por embargo na Amazônia

**Teste:** Embargo na Amazônia adiciona pontos extras

### AC-04: Adicionar peso por área embargada
- **Dado** que um embargo possui área embargada (`QTD_AREA_EMBARGADA`)
- **Quando** o sistema calcula o score de risco
- **Então** deve adicionar **+1 ponto** por cada 10 hectares embargados (máximo +10)

**Teste:** 100+ hectares adicionam 10 pontos extras

### AC-05: Limitar score máximo a 100
- **Dado** que os fatores somam mais de 100 pontos
- **Quando** o sistema calcula o score de risco
- **Então** deve limitar o score a 100

**Teste:** Score nunca excede 100

### AC-06: Classificar nível de risco como Crítico
- **Dado** que o score calculado é >= 80
- **Quando** o sistema classifica o nível de risco
- **Então** o campo `riskLevel` deve ser "Crítico"

**Teste de Contrato:** `a request to search entity with critical risk` com state `an entity with critical risk level exists`

### AC-07: Retornar score e riskLevel na resposta
- **Dado** que a entidade foi encontrada
- **Então** a resposta deve conter:
  - `entity.score`: Integer - pontuação calculada (0-100)
  - `entity.riskLevel`: String - "Baixo", "Médio", "Alto" ou "Crítico"

**Teste:** Validação dos campos na resposta JSON

## Regras de Negócio

### RN-01: Algoritmo de Cálculo do Score

```
score = 0

Para cada embargo:
  score += 10                                    // Base por embargo

  Se SIT_DESMATAMENTO = 'D':
    score += 5                                   // Peso desmatamento

  Se COD_TIPO_BIOMA = 4 (Amazônia):
    score += 3                                   // Peso bioma crítico

  score += min(QTD_AREA_EMBARGADA / 10, 10)     // Peso área (max +10)

score = min(score, 100)                         // Limite máximo
```

### RN-02: Exemplo de Cálculo (Risco Crítico)

Entidade com 5 embargos, todos com desmatamento na Amazônia, 200 hectares cada:

```
Embargo 1: 10 + 5 + 3 + 10 = 28
Embargo 2: 10 + 5 + 3 + 10 = 28
Embargo 3: 10 + 5 + 3 + 10 = 28
Embargo 4: 10 + 5 + 3 + 10 = 28
Embargo 5: 10 + 5 + 3 + 10 = 28
Total: 140 → limitado a 100
```

### RN-03: Exemplo de Cálculo (Risco Baixo)

Entidade com 1 embargo, sem desmatamento, Mata Atlântica, 5 hectares:

```
Embargo 1: 10 + 0 + 0 + 0 = 10
Total: 10 (Baixo)
```

|---------|----------------|
| 0-25    | Baixo          |
| 26-50   | Médio          |
| 51-79   | Alto           |
| 80-100  | Crítico        |

@State("an entity with critical risk level exists")
void setupEntityWithCriticalRisk() {
    QuarkusTransaction.requiringNew().run(() -> {
        EmbargoRepository repository = getRepository();
        repository.deleteAll();

            embargo.setSeqTad(8L + i);

            embargo.setSerTad("A");
            embargo.setNomePessoaEmbargada("Mineradora Vermelha S.A.");
            embargo.setCpfCnpjEmbargado("44555666000181");
            embargo.setTpAreaEmbargada("Desmatamento");
            embargo.setDatEmbargo(LocalDateTime.of(2024, 6, 1 - i, 0, 0));
            embargo.setDesTad("Embargo por desmatamento ilegal");
            embargo.setSigUfTad("PA");
            embargo.setNomMunicipioTad("Altamira");
            embargo.setSitDesmatamento("D");                    // +5 pontos
            embargo.setCodTipoBioma(4);                          // Amazônia +3 pontos
            embargo.setDesTipoBioma("Amazônia");
            embargo.setQtdAreaEmbargada(BigDecimal.valueOf(150)); // +10 pontos
            embargo.setNumAutoInfracao("45678" + i);
            embargo.setSerAutoInfracao("A");

            repository.persist(embargo);
        }
        // Total: 5 embargos × (10 + 5 + 3 + 10) = 140 → limitado a 100
    });
}

### Matchers do Contrato

O contrato usa `"match": "integer"` para o score, permitindo que o valor real seja diferente do exemplo (89), desde que seja um inteiro. O `riskLevel` usa `"match": "type"`, permitindo qualquer string.

**Importante:** O teste de contrato valida a estrutura, não os valores exatos. A implementação deve garantir que:
- `score` seja >= 80 para que `riskLevel` seja "Crítico"
