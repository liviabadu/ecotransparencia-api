# US-007: Validar Status Cadastral do CNPJ na Receita Federal

## Descrição

**Como** usuário do sistema EcoTransparência
**Quero** que o sistema valide o status cadastral do CNPJ na Receita Federal antes de realizar a análise
**Para que** eu tenha certeza de que estou analisando apenas empresas com situação cadastral ATIVA

## Contexto

Antes de qualquer consulta às bases de dados internas (embargos, autos de infração), o sistema deve consultar uma API externa para validar o status cadastral de um CNPJ na Receita Federal. Se o CNPJ estiver **ATIVO**, a análise prossegue normalmente. Caso contrário, o sistema deve retornar uma mensagem informando o status e não realizar a análise ASG.

Esta validação é aplicável apenas para CNPJs. Para CPFs, a validação de formato já existente é suficiente.

## Endpoints Afetados

```
GET /api/search/document?document={cnpj}&type=cnpj
```

---

## Critérios de Aceitação

### AC-01: CNPJ com situação cadastral ATIVA

- **Dado** que o usuário informa um CNPJ válido
- **Quando** o sistema consulta a API da Receita Federal
- **E** a situação cadastral retornada é "ATIVA"
- **Então** o sistema deve prosseguir com a consulta às bases de dados (embargos, autos de infração)
- **E** retornar o Score ASG normalmente

**Teste:** `shouldProceedWithAnalysisWhenCnpjIsActive`

### AC-02: CNPJ com situação cadastral diferente de ATIVA

- **Dado** que o usuário informa um CNPJ válido
- **Quando** o sistema consulta a API da Receita Federal
- **E** a situação cadastral retornada é diferente de "ATIVA" (ex: BAIXADA, SUSPENSA, INAPTA, NULA)
- **Então** o sistema deve retornar uma resposta indicando o status do CNPJ
- **E** o campo `bloqueadoPorSituacaoCadastral` deve ser `true`
- **E** não deve prosseguir com a análise ASG (não consulta bases de embargos/autos)

**Teste:** `shouldBlockAnalysisWhenCnpjIsNotActive`

### AC-03: Falha na consulta à API da Receita Federal

- **Dado** que o usuário informa um CNPJ válido
- **Quando** o sistema tenta consultar a API da Receita Federal
- **E** ocorre uma falha (timeout, indisponibilidade, erro HTTP)
- **Então** o sistema deve retornar uma mensagem de erro apropriada
- **E** não deve prosseguir com a análise ASG (fail-safe)

**Teste:** `shouldBlockAnalysisWhenReceitaFederalApiUnavailable`

### AC-04: CNPJ com formato inválido

- **Dado** que o usuário informa um CNPJ com formato inválido (dígitos verificadores incorretos)
- **Quando** o sistema valida o CNPJ
- **Então** o sistema deve retornar erro de validação
- **E** não deve consultar a API da Receita Federal

**Teste:** `shouldRejectInvalidCnpjWithoutCallingApi`

### AC-05: Consulta com CPF (comportamento inalterado)

- **Dado** que o usuário informa um CPF válido
- **Quando** o sistema processa a consulta
- **Então** a validação de situação cadastral não deve bloquear a análise
- **E** o sistema deve prosseguir normalmente com a análise ASG

**Teste:** `shouldProceedWithCpfAnalysisWithoutCnpjValidation`

## Regras de Negócio

### RN-01: Validação prévia obrigatória para CNPJ

- Toda consulta por CNPJ **deve** passar pela validação de situação cadastral **antes** de consultar as bases de embargos e autos de infração
- A validação deve ocorrer mesmo que o CNPJ não tenha ocorrências nas bases

### RN-02: Situação ATIVA como única permissão

- Apenas CNPJs com situação cadastral **"ATIVA"** podem ter a análise ASG realizada
- Todas as demais situações (BAIXADA, SUSPENSA, INAPTA, NULA) bloqueiam a análise

### RN-03: Fail-safe em caso de falha

- Se a API externa estiver indisponível ou retornar erro, a análise **não deve prosseguir**
- O sistema deve informar ao usuário que não foi possível validar o CNPJ

### RN-04: CPF não afetado

- Esta validação de bloqueio **não se aplica** a consultas por CPF
- Para CPF, apenas a validação de formato (dígitos verificadores) é realizada

### RN-05: Cache de consultas (recomendado)

- Consultas à API externa podem ser cacheadas por 1-24 horas para evitar chamadas repetidas
- O cache deve ser invalidado se houver necessidade de revalidação explícita

## Notas Técnicas

### API Externa Utilizada

**CNPJA API Open** - API pública e gratuita sem necessidade de autenticação:
- Documentação: https://cnpja.com/api/open

```bash
# Exemplo de request
curl --request GET --url 'https://open.cnpja.com/office/{cnpj}'
```

Resposta relevante:
```json
{
  "taxId": "07526557011659",
  "company": {
    "name": "AMBEV S.A."
  },
  "status": {
    "id": 2,
    "text": "Ativa"
  }
}
```

**Valores de status.id:**
- `1` = Nula
- `2` = Ativa
- `3` = Suspensa
- `4` = Inapta
- `8` = Baixada

### Configuração

```properties
# application.properties
ecotransparencia.receita-federal.use-real-api=true
quarkus.rest-client.cnpja-api.url=https://open.cnpja.com
quarkus.rest-client.cnpja-api.connect-timeout=5000
quarkus.rest-client.cnpja-api.read-timeout=10000
```

### Timeout e Resiliência

- Timeout recomendado: 5-10 segundos
- Retry: 1 tentativa adicional em caso de falha
- Circuit breaker: Considerar para produção
